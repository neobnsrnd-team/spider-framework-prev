# CMS 역할 및 권한 정책

## 개요

현재 CMS 기능은 제작자 화면과 관리자 화면이 분리되어 있으며, 권한도 `CMS:R`, `CMS:W` 기준으로 나누어 관리한다.

이 문서는 현재 확정된 역할 정책과 화면 접근 범위, 허용 동작, 상태별 제약을 정리한다.

## 역할 정의

### CMS:R

일반 제작자 권한이다.

주요 성격:

- 제작자용 화면 사용
- 본인 페이지만 조회 및 작업

### CMS:W

관리자/결재자 권한이다.

주요 성격:

- 관리자 화면 사용
- 전체 페이지 관리 및 승인 처리

## 화면 접근 정책

### CMS:R 접근 가능 화면

- `/cms/dashboard`
- 본인 페이지에 대한 `/cms/edit`

### CMS:R 접근 불가 화면

- `/cms-admin/**`

### CMS:W 접근 가능 화면

- `/cms-admin/**`

### CMS:W 접근 불필요 화면

- `/cms/dashboard`

현재 정책상 `/cms/dashboard`는 일반 유저용 화면으로 본다. 따라서 `CMS:W`는 기본적으로 `/cms/dashboard` 접근이 필요하지 않다.
그러나 CMS:W 권한은 R+W의 권한을 가지므로 `/cms/dashboard`에 대한 권한은 포함하고 있다.

## 권한별 허용 기능

### CMS:R 허용 기능

- 본인 페이지 조회
- 본인 페이지 생성
- 본인 페이지 `/cms/edit` 접근
- 본인 페이지 수정
- 본인 페이지 저장
- 본인 페이지 삭제
- 본인 페이지 승인 요청

### CMS:R 제한 기능

- 다른 사용자의 페이지 조회
- 다른 사용자의 페이지 수정
- 다른 사용자의 페이지 저장
- 다른 사용자의 페이지 삭제
- 다른 사용자의 페이지 승인 요청
- 승인
- 반려
- 상위 승인 요청
- 관리자 기능 수행

### CMS:W 허용 기능

- 모든 페이지 조회
- 전체 페이지 관리
- 승인
- 반려
- 상위 승인 요청
- 관리자 화면에서 필요한 페이지 직접 수정 기능
- 추후 관리자 템플릿 생성 등 관리자 제작 기능

### CMS:W 제한 사항

- 일반 유저용 `/cms/dashboard` 사용은 기본 정책 범위에서 제외

## 페이지 소유권 정책

`CMS:R` 사용자는 본인이 생성한 페이지에 대해서만 작업할 수 있다.

적용 대상:

- 목록 조회
- `/cms/edit` 접근
- 수정
- 저장
- 삭제
- 승인 요청

즉, `CMS:R`는 본인 페이지에만 작업 가능하고, `CMS:W`는 전체 페이지에 작업 가능하다.

## 승인 상태 정책

### PENDING 상태

현재 확정 정책은 기존 구현과 동일하게 유지한다.

- `PENDING` 상태에서는 재승인요청 불가

즉, 승인 요청 가능 상태는 다음과 같다.

- `WORK`
- `REJECTED`
- `APPROVED`

다음 상태에서는 재승인요청이 불가능하다.

- `PENDING`

## 삭제 정책

### CMS:R

- 본인이 만든 페이지에 대해서만 삭제 가능

### CMS:W

- 모든 페이지 삭제 가능

## 정책 요약

### CMS:R

- 제작자 권한
- `/cms/dashboard` 사용
- 본인 페이지만 조회 가능
- 본인 페이지만 생성/편집/저장/삭제/승인요청 가능
- `PENDING` 상태에서는 재승인요청 불가

### CMS:W

- 관리자/결재자 권한
- `/cms-admin/**` 사용
- 모든 페이지 조회 가능
- 승인/반려/상위 승인 요청 가능
- 전체 페이지 관리 가능
- 필요 시 직접 수정 가능

## 구현 시 주의사항

- 화면에서 버튼을 숨기는 것만으로 권한 제어를 끝내면 안 된다.
- 서버 API에서도 동일한 권한/소유권 검증이 필요하다.
- `CMS:R`의 본인 페이지 제한은 UI와 서버 양쪽에서 함께 보장해야 한다.
- `PENDING` 상태 재승인요청 금지는 UI와 서버 양쪽에서 함께 보장해야 한다.
