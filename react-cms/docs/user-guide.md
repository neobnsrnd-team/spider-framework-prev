## 1. 빌더 화면 구성

빌더는 세 영역으로 나뉩니다.

```
┌──────────────────────────────────────────────────────────────┐
│  툴바 (가져오기 · 내보내기 · 코드 보기 · 페이지 저장 · 미리보기)  │
├─────────────┬────────────────────────────┬───────────────────┤
│             │                            │                   │
│  좌측 패널  │       중앙 캔버스           │   우측 패널       │
│  블록 탭    │  레이아웃 크롬(헤더/푸터)   │  속성 탭          │
│  오버레이 탭│  + 블록들                  │  레이아웃 탭      │
│             │                            │  JSON 탭          │
└─────────────┴────────────────────────────┴───────────────────┘
```

| 영역 | 역할 |
|---|---|
| 좌측 패널 | 블록 목록(블록 탭) 및 오버레이 목록(오버레이 탭) |
| 중앙 캔버스 | 실제 페이지 편집 화면. 블록을 드래그&드롭으로 배치 |
| 우측 패널 | 선택한 블록의 속성 편집(속성 탭), 레이아웃 설정(레이아웃 탭), 현재 페이지 JSON(JSON 탭) |

## 2. 레이아웃 선택

레이아웃은 페이지의 헤더·푸터 구조를 결정합니다. 블록을 추가하기 전에 먼저 설정하는 것을 권장합니다.

1. 우측 패널 상단에서 **레이아웃** 탭을 클릭합니다.
2. 드롭다운에서 레이아웃을 선택합니다.

| 레이아웃 | 설명 |
|---|---|
| **Home Page Layout** | 홈 헤더 + 콘텐츠 + 하단 탭 바(BottomNav) |
| **Page Layout** | 뒤로가기 헤더 + 콘텐츠 + 선택적 하단 버튼 |
| **Blank Layout** | 헤더·푸터 없음 |

3. 선택 후 레이아웃별 속성(타이틀, 하단 버튼 수 등)을 편집합니다.
4. 레이아웃을 선택하지 않아도 블록만 배치할 수 있습니다.

## 3. 블록 추가하기

### 드래그&드롭

1. 좌측 패널의 **블록** 탭에서 원하는 블록을 찾습니다.
2. 상단 검색창에 블록 이름을 입력하면 빠르게 찾을 수 있습니다.
3. 블록을 클릭한 채로 중앙 캔버스로 드래그합니다.
4. 원하는 위치에서 손을 뗍니다.

### 블록 카테고리

| 카테고리 | 설명 |
|---|---|
| **core** | Button, Input, Typography 등 기본 UI 요소 |
| **biz** | 계좌 카드, 카드 결제 요약 등 금융 도메인 특화 컴포넌트 |
| **modules** | 섹션 헤더, 탭, 날짜 선택기 등 복합 UI 모듈 |

biz·modules 카테고리는 도메인(common / banking / card / insurance)으로 세분화되어 있습니다.

## 4. 블록 속성 편집

1. 캔버스에서 블록을 클릭하면 선택(파란 테두리)됩니다.
2. 우측 패널이 자동으로 **속성** 탭으로 전환됩니다.
3. 각 필드를 편집합니다.

### 속성 필드 유형

| 유형 | UI | 사용 예 |
|---|---|---|
| 텍스트 | 텍스트 입력창 | 버튼 레이블, 타이틀 |
| 숫자 | 숫자 입력창 | 금액, 단계 수 |
| 토글 | 켜기/끄기 스위치 | 전체 너비, 비활성화 여부 |
| 드롭다운 | 선택 목록 | 변형(variant), 크기(size) |
| 아이콘 선택기 | 아이콘 피커 | 아이콘 종류 |
| 그룹 | 접을 수 있는 하위 필드 묶음 | 레이블 텍스트·크기 묶음 |
| 목록 | 항목 추가·삭제 가능한 리스트 | 탭 항목, 메뉴 항목 |
| 액션 | 이벤트 연결 선택기 | 클릭 시 동작 |

### 패딩 조정

속성 패널 하단에서 블록의 상·하·좌·우 패딩을 각각 조정할 수 있습니다 (none / xs / sm / md / lg / xl).

## 5. 블록 순서 변경 · 삭제

### 순서 변경

캔버스에서 블록을 드래그하여 다른 블록 위아래로 이동할 수 있습니다.

### 삭제

블록을 선택하면 나타나는 컨트롤 버튼에서 **삭제(✕)** 를 클릭합니다.

### 전체 초기화

툴바 우측의 **초기화** 버튼을 클릭하면 캔버스의 모든 블록과 레이아웃이 제거됩니다.

## 6. 오버레이 추가 · 편집

오버레이는 바텀시트·모달 등 페이지 위에 표시되는 팝업 영역입니다.

### 오버레이 추가

1. 좌측 패널 상단에서 **오버레이** 탭을 클릭합니다.
2. **+ 추가** 버튼을 클릭하고 오버레이 종류를 선택합니다.

| 오버레이 | 설명 |
|---|---|
| 바텀시트 | 하단에서 올라오는 시트. 내부에 블록 자유 배치 가능 |
| 바텀시트 (텍스트 구성) | 텍스트 내용만 입력하는 간단한 바텀시트 |
| 모달 | 화면 중앙 팝업. 내부에 블록 자유 배치 가능 |
| 모달 (텍스트 구성) | 텍스트 내용만 입력하는 간단한 모달 |
| 이용내역 검색 필터 | 카드 이용내역 검색 조건 필터 시트 |
| PIN 입력 | 비밀번호 입력 시트 |
| 슬라이드 오버 | 옆에서 밀려오는 슬라이드 패널 |

### 오버레이 내 블록 편집

1. 오버레이 목록에서 편집할 오버레이를 클릭합니다.
2. 캔버스가 오버레이 편집 모드로 전환됩니다 (툴바에 오버레이 이름 표시).
3. 일반 블록과 동일하게 드래그&드롭으로 블록을 추가·편집합니다.
4. 툴바의 **← 페이지** 버튼을 클릭하면 페이지 편집으로 돌아옵니다.

### 오버레이 속성 편집

오버레이 편집 모드에서 우측 패널의 **오버레이** 탭에서 제목, 버튼 수 등을 편집할 수 있습니다.

## 7. 블록에 액션 연결하기

블록의 클릭 이벤트에 동작을 연결할 수 있습니다.

1. 블록을 선택해 우측 **속성** 탭을 엽니다.
2. `클릭`, `탭 변경` 등 이벤트 필드에서 액션을 선택합니다.

| 액션 | 설명 |
|---|---|
| **openOverlay** | 특정 오버레이 열기 (오버레이 ID 선택) |
| **closeOverlay** | 현재 오버레이 닫기 |
| **navigate** | 특정 경로로 페이지 이동 (경로 직접 입력) |

> 오버레이를 먼저 추가해야 `openOverlay` 액션에서 선택할 수 있습니다.

---

## 8. 미리보기

툴바의 **미리보기** 버튼을 클릭하면 새 탭에서 현재 페이지 전체 화면을 확인할 수 있습니다.

- 저장하지 않아도 현재 편집 상태 그대로 미리보기가 열립니다.
- 블록이 하나 이상 있어야 버튼이 활성화됩니다.

## 9. 페이지 저장

툴바의 **페이지 저장** 버튼을 클릭합니다.

### 저장 모달 입력 항목

**Admin 연동 모드**

| 항목 | 설명 | 예시 |
|---|---|---|
| 컴포넌트명 | PascalCase 영문·숫자 | `MyDashboardPage` |

**단독 실행 모드** (추가 항목)

| 항목 | 설명 | 예시 |
|---|---|---|
| 컴포넌트명 | PascalCase 영문·숫자 | `MyDashboardPage` |
| 라우트 경로 | `/`로 시작하는 URL 경로 | `/my-dashboard` |

### 저장 결과

- **Admin 연동 모드**: Oracle DB에 저장됩니다. 저장 후 승인 요청을 별도로 진행해야 합니다.
- **단독 실행 모드**: `demo/front` 앱에 JSX 파일이 생성되고 라우트가 자동 등록됩니다.

### 재저장 (수정)

동일한 컴포넌트명으로 저장하면 새로 생성하지 않고 기존 페이지를 덮어씁니다.

### APPROVED 상태에서 저장 시 주의

승인 완료된 페이지를 수정하면 승인이 취소되고 '작업 중(WORK)' 상태로 돌아갑니다.  
저장 버튼 클릭 시 확인 팝업이 표시됩니다.

## 10. 승인 상태 확인

Admin 연동 모드에서는 페이지마다 승인 상태가 관리됩니다.

| 상태 | 표시 | 설명 |
|---|---|---|
| **WORK** | (배너 없음) | 작업 중 — 자유롭게 편집·저장 가능 |
| **PENDING** | 노란 배너 | 승인 요청 중 — 저장 버튼 비활성화. 승인 완료 또는 요청 취소 후 수정 가능 |
| **APPROVED** | (배너 없음) | 승인 완료 — 수정 시 WORK 상태로 되돌아감 |
| **REJECTED** | 빨간 배너 | 반려 — 배너에 반려 사유 표시. 수정 후 재저장하여 다시 요청 가능 |

## 11. 기존 페이지 불러와 수정하기

Admin 연동 모드에서는 대시보드에서 기존 페이지를 선택해 빌더로 진입할 수 있습니다.

- 빌더 URL에 `?pageId=xxx` 파라미터가 붙어 있으면 자동으로 해당 페이지를 불러옵니다.
- 편집 모드에서는 툴바의 **가져오기** 버튼(외부 JSON 파일 불러오기)이 표시되지 않습니다.  
  기존 편집 내용을 실수로 덮어쓰는 것을 방지하기 위함입니다.

### 툴바 부가 기능

| 버튼 | 설명 |
|---|---|
| 가져오기 | 로컬 JSON 파일을 불러와 캔버스에 적용 (신규 생성 모드에서만 표시) |
| 내보내기 | 현재 캔버스 상태를 JSON 파일로 다운로드 |
| 코드 보기 | 현재 페이지의 생성된 React JSX 코드를 팝업으로 확인 및 복사 |
