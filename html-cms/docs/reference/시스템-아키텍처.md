# 시스템 아키텍처

> 작성일: 2026-05-06
> 대상 프로젝트: `html-cms` (CMS), `spider-admin` (관리자 콘솔)

---

## 한 줄 요약

**현업 관리자가 페이지·이미지를 만들고(CMS) → 결재자가 승인하고(Spider Admin) → 일반 사용자에게 배포(CMS Operation)하는 3-tier 구조다.**

DB·이미지 저장소를 공유하지만, 권한·역할·배포 경로는 인프라 레벨에서 분리되어 있다.

---

## 전체 아키텍처

```
                                    [사용자 브라우저]
                                           │
                ┌──────────────────────────┼──────────────────────────┐
                │                          │                          │
                ▼                          ▼                          ▼
       ┌────────────────┐         ┌────────────────┐         ┌────────────────┐
       │  Spider Admin  │         │   CMS :3000    │         │   CMS :3001    │
       │  (관리자 콘솔) │         │  (cms 모드)    │         │ (operation 모드)│
       │                │         │                │         │                │
       │  Spring Boot   │         │   Next.js      │         │   Next.js      │
       │  Thymeleaf     │         │   React        │         │   React        │
       │  Spring Sec    │         │ ContentBuilder │         │ ContentBuilder │
       └───────┬────────┘         └───────┬────────┘         └───────┬────────┘
               │                          │                          │
               │  REST API                │                          │
               │  multipart/form-data     │                          │
               │  x-deploy-token          │                          │
               └─────────────►◄───────────┘                          │
                                          │                          │
                                          │   배포 push/receive       │
                                          └─────────►◄───────────────┘
                │                          │                          │
                │                          │                          │
                ▼                          ▼                          ▼
       ┌──────────────────────────────────────────────────────────────┐
       │                    Oracle DB (공유)                          │
       │  spider-admin 스키마 + html-cms 스키마 (FWK_*, SPW_CMS_*)    │
       └──────────────────────────────────────────────────────────────┘
                                          │
                                          ▼
       ┌──────────────────────────────────────────────────────────────┐
       │                     호스트 파일시스템                        │
       │  /data/uploads/  (미승인 이미지 — :3000 쓰기, :3001 읽기)    │
       │  /data/deployed/ (배포 페이지·승인 이미지 — :3001 쓰기)      │
       └──────────────────────────────────────────────────────────────┘
```

## 컴포넌트별 상세

### 1. Spider Admin (`spider-admin/`)

**기술 스택**
- Spring Boot 3.4.7 / Java 17
- Thymeleaf (SSR) + Bootstrap 5.3 + jQuery
- MyBatis 3.0.3 + Oracle/MySQL
- Spring Security + JWT
- 기본 포트: 8080

**역할**
- 관리자/결재자 로그인·권한·역할 관리
- 콘텐츠 승인·반려 UI
- CMS API 호출로 이미지·페이지 승인 처리
- 작업 이력 로그
- 미들웨어(WAS·배치·메시지 인스턴스) 관리

**주요 디렉토리**
```
spider-admin/src/main/java/com/example/spideradmin/
├── domain/
│   ├── cmsasset/        # CMS 이미지 자산 관리 (업로드 중계, 승인)
│   ├── cmspage/         # CMS 페이지 관리
│   ├── wasinstance/     # WAS 인스턴스 관리
│   └── ...
├── global/              # 공통 인프라 (security, config, exception)
└── infra/               # TCP, HTTP 클라이언트
```

---

### 2. CMS (`html-cms/`)

**기술 스택**
- Next.js 15.5.4 (App Router) / React 19 / TypeScript
- Tailwind CSS 4
- ContentBuilder.js (비주얼 에디터)
- oracledb 드라이버 (Oracle 직접 연결)
- node-cron (만료 스케줄러)
- 기본 포트: 3000 (동일 이미지로 두 컨테이너 실행)

**역할**
- 페이지 비주얼 에디터 (`/edit`)
- 페이지 뷰어 (`/view`)
- 페이지·에셋 CRUD API
- A/B 테스트 라우팅
- 페이지뷰·클릭 추적
- 배포된 HTML 정적 서빙

**주요 디렉토리**
```
html-cms/src/
├── app/api/             # REST API (builder/, assets/, deploy/, ab/, track/)
├── components/          # React 컴포넌트 (edit/, view/, dashboard/, approve/)
├── db/                  # Oracle DB 레이어 (repository, queries, ddl)
└── lib/                 # 유틸·인증·환경변수
```

**두 컨테이너 — 동일 이미지, 다른 환경변수**

같은 Docker 이미지로 두 컨테이너를 실행하고, `SERVER_MODE` 환경변수로 동작을 분기한다.

| | CMS :3000 (cms 모드) | CMS :3001 (operation 모드) |
|---|---|---|
| 대상 사용자 | 현업 관리자 | 현업 제작자 + 일반 사용자 |
| 이미지 업로드 | 가능 | OS 레벨 차단 (`:ro` 마운트) |
| 페이지 배포 수신 | — | `/deploy/receive` 활성 |
| 정적 HTML 서빙 | — | `/data/deployed/{pageId}.html` |

자세한 설계 이유는 [`서버-포트-분리-설계.md`](./서버-포트-분리-설계.md) 참고.

---

### 3. Oracle DB (공유)

두 프로젝트가 같은 Oracle 인스턴스를 사용한다. 스키마는 분리되어 있지만 일부 테이블은 양쪽에서 참조한다.

**spider-admin 영역**
- `FWK_USER`, `FWK_ROLE`, `FWK_MENU` — 사용자·권한·메뉴
- `FWK_WAS_INSTANCE` — WAS 인스턴스
- `FWK_CMS_FILE_SEND_HIS` — CMS 배포 이력 수신
- `FWK_CMS_SERVER_INSTANCE` — CMS 서버 인스턴스 (A/B 라우팅용)

**html-cms 영역 (`SPW_CMS_*`)**
- `SPW_CMS_PAGE` — 페이지 (HTML CLOB 포함)
- `SPW_CMS_ASSET` — 이미지 에셋 메타데이터
- `SPW_CMS_PAGE_HISTORY` — 페이지 변경 이력
- `SPW_CMS_AB_GROUP`, `SPW_CMS_AB_PAGE` — A/B 테스트
- `SPW_CMS_PAGE_VIEW_LOG`, `SPW_CMS_CLICK_LOG` — 추적 로그

---

### 4. 파일시스템 저장소

이미지·배포 HTML은 DB가 아닌 호스트 파일시스템에 물리 파일로 저장된다.

```
/data/
├── uploads/             # 미승인 이미지 보관소
│   └── {assetId}_{원본파일명}
└── deployed/
    ├── img/             # 승인된 이미지
    │   └── {assetId}_{원본파일명}
    └── {pageId}.html    # 배포된 페이지 HTML
```

Docker 볼륨 바인드 마운트로 컨테이너에서 접근:

```yaml
cms (:3000):
  - /data/uploads:/app/public/uploads        # 쓰기
  - /data/deployed:/app/public/deployed

operation (:3001):
  - /data/uploads:/app/public/uploads:ro     # 읽기 전용
  - /data/deployed:/app/public/deployed      # 쓰기
```

---

### 5. nginx (리버스 프록시)

```
인터넷 ──► nginx :8080 ──► CMS :3001 (대고객 + 제작자)
사내망 ──► nginx :80   ──► CMS :3000 (현업 관리자)
사내망 ──► spider-admin :8080 (Tomcat 직접)
```

nginx가 추가로 처리하는 것:
- 정적 파일 서빙 (`/static/` → `/data/uploads/`, `/deployed/static/` → `/data/deployed/img/`)
- 트래커 API CORS 처리 (배포된 페이지에서 크로스오리진으로 호출)
- 배포된 HTML 직접 서빙 (`/cms/deployed/` → Next.js 우회, `/data/deployed/`에서 직접 읽음)

## 통신 구조

### 사용자 → 시스템

| 사용자 | 접속 경로 | 주요 작업 |
|---|---|---|
| 현업 관리자 | `nginx :80` → CMS :3000 | 이미지 업로드, 빈 화면 템플릿 제작 |
| 결재자 | spider-admin :8080 | 승인·반려 |
| 현업 제작자 | `nginx :8080` → CMS :3001 | 템플릿 기반 페이지 제작 |
| 일반 사용자 | `nginx :8080` → CMS :3001 | 배포된 콘텐츠 열람 |

### Spider Admin ↔ CMS (서버 간)

REST API + multipart/form-data 방식으로 통신한다. 서버 간 호출은 `x-deploy-token` 헤더로 인증한다.

```
Spider Admin                    CMS
     │                          │
     │  POST /cms/api/builder/upload  (이미지 업로드 중계)
     │─────────────────────────►│
     │                          │
     │  POST /cms/api/assets/{id}/deploy  (이미지 승인 후 파일 이동 트리거)
     │─────────────────────────►│
     │                          │
     │  POST /cms/api/builder/pages/{id}/approve  (페이지 승인)
     │─────────────────────────►│
     │                          │
     │  POST /cms/api/deploy/push  (페이지 배포 트리거)
     │─────────────────────────►│
     │                          │
```

자세한 통신 방식·인증·보상 트랜잭션은 별도 문서(예정).

### CMS 내부 (cms ↔ operation)

```
CMS :3000 (cms)              CMS :3001 (operation)
       │                              │
       │  POST /deploy/push           │
       │  배포 트리거                  │
       │─────────────────────────────►│
       │                              │  /data/deployed/{pageId}.html 생성
       │                              │  FWK_CMS_FILE_SEND_HIS UPSERT
       │                              │  (Spider Admin DB에 배포 이력 전달)
       │  { ok: true }                │
       │◄─────────────────────────────│
```

## 인증·보안 구조

### 사용자 인증
- spider-admin: Spring Security + JWT 발급, 세션 쿠키로 유지
- CMS: spider-admin이 발급한 JWT를 검증 (`fetchJavaAdminApi('/api/auth/me')`)

### 서버 간 인증
- `x-deploy-token` 헤더 (공유 시크릿, `DEPLOY_SECRET` 환경변수)
- `timingSafeEqual()`로 비교 (타이밍 공격 방지)

### 권한 체계 (현재)
- `cms_admin` — 쓰기 가능 (페이지·에셋 생성·수정)
- `cms_user` — 읽기만 (열람)
- 향후: 현업 관리자 / 결재자 / 현업 제작자 3단계로 세분화 예정

### 파일 접근 격리
- `/data/uploads/`는 :3001에 `:ro` 읽기 전용 마운트 → OS 레벨로 미승인 이미지 업로드 차단

## 배포 환경

### 컨테이너 구성 (`html-cms/docker-compose-prod.yml`)

```
[NHN Cloud 인스턴스]
├── springware-cms        (CMS :3000, SERVER_MODE=cms)
├── springware-operation  (CMS :3001, SERVER_MODE=operation)
└── nginx                 (리버스 프록시)

[별도 인스턴스 또는 동일 호스트]
└── spider-admin (Spring Boot, :8080)

[외부 인스턴스]
└── Oracle DB (개발 DB 공용)
```

### CI/CD
- spider-admin: GitHub Actions → Build (Spotless+JaCoCo) → E2E (Playwright) → SonarCloud
- html-cms: NHN Cloud + GitHub Actions, Docker 이미지 푸시 → 운영 인스턴스 pull & restart

## 시스템 경계 정리

| 경계 | 어디서 끝나고 어디서 시작하나 |
|---|---|
| 사용자 ↔ 서버 | nginx 리버스 프록시 (포트별 라우팅) |
| Spider Admin ↔ CMS | REST API + `x-deploy-token` |
| CMS :3000 ↔ :3001 | 공유 DB + 공유 파일시스템 (`/data/`) + 배포 HTTP 호출 |
| 애플리케이션 ↔ DB | oracledb 드라이버 (Next.js) / MyBatis (Spring Boot) |
| 애플리케이션 ↔ 파일 | Docker 볼륨 바인드 마운트 (`:ro` 권한 분리) |
